**Overview**
- **Modello:** Meta `Llama 3 70B Instruct`
- **Perché:** Buon ragionamento per pianificazione strutturata.
- **Output:** Piano in YAML con sezioni coerenti.

**Use Case**
- **Scenario:** Devi migrare un'API legacy verso un nuovo stack. Vuoi un piano dettagliato e ordinabile.

**Prompt**

```
Ruolo: Sei un tech lead. Crea un piano di migrazione eseguibile.

Output SOLO YAML con:
version: 1
obiettivi:
  - ...
attivita:
  - id: M1
    descrizione: ...
    dipendenze: [ ]
    owner: ruolo/team
    durata_giorni: 0
rischi:
  - id: R1
    descrizione: ...
    mitigazione: ...
metriche_successo:
  - ...

Input contesto (<ctx>):
<ctx>
{{CONTESTO_MIGRAZIONE}}
</ctx>
```

**Perché Funziona**
- **Formato YAML:** facile da tracciare/versionare.
- **Campi minimi:** permettono stima e Dipendenze chiare.

**Fonti**
- Meta/Llama — suggerimenti generali per prompting istruttivo.
- Pattern di project planning con output strutturati.
