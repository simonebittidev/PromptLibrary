**Overview**
- **Modello:** OpenAI `gpt-4.1-mini`
- **Perché:** Economico e veloce per estrazioni strutturate ad alta precisione.
- **Output:** JSON aderente a uno schema target predefinito.

**Use Case**
- **Scenario:** Arrivano ticket via email/chat. Serve normalizzare `categoria`, `priorità`, `cliente`, `prodotto`, `descrizione`, `riproducibilità`, `allegati` in un JSON coerente.

**Prompt**

```
Ruolo: Sei un estrattore di dati accurato. Devi mappare un testo di ticket a un JSON valido secondo uno schema.

Schema target (JSON Schema-like):
{
  "type": "object",
  "required": ["categoria", "priorita", "cliente", "prodotto", "descrizione"],
  "properties": {
    "categoria": {"enum": ["bug", "richiesta", "domanda", "altro"]},
    "priorita": {"enum": ["bassa", "media", "alta", "critica"]},
    "cliente": {"type": "string"},
    "prodotto": {"type": "string"},
    "descrizione": {"type": "string"},
    "riproducibilita": {"enum": ["sempre", "spesso", "intermittente", "non_riproducibile", "nd"]},
    "allegati": {"type": "array", "items": {"type": "string", "format": "uri"}}
  }
}

Istruzioni:
- Leggi il ticket tra <ticket></ticket>.
- Se un campo è assente, usa una value valida (es. "nd" per riproducibilità) e non inventare link/allegati.
- Restituisci SOLO il JSON, nessun testo aggiuntivo.

Input:
<ticket>
{{TESTO_TICKET}}
</ticket>
```

**Perché Funziona**
- **Schema vincolante:** riduce variabilità e facilita validazione automatica.
- **Enum espliciti:** guidano la classificazione a classi chiuse.
- **Delimitazione input:** previene confusione modello/istruzioni.
