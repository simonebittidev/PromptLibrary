**Overview**
- **Modello:** xAI `Grok‑1.5`
- **Perché:** Buon reasoning e concisione per definire regole operative.
- **Output:** Regole alert in JSON pronte per l’uso.

**Use Case**
- **Scenario:** Vuoi definire alert da log grezzi. Serve un set di regole con severità, pattern e hint di remediation.

**Prompt**

```
Ruolo: Sei un ingegnere di affidabilità. Crea regole di alert partendo da log.

Output SOLO JSON:
{
  "regole": [
    {
      "id": "ALR-001",
      "descrizione": "...",
      "severita": "low|medium|high|critical",
      "pattern": "regex o substring",
      "finestra_min": 5,
      "soglia": 10,
      "azione": "pagina_oncall|ticket|log_only",
      "remediation_hint": "..."
    }
  ]
}

Istruzioni:
- Deduplica pattern equivalenti.
- Evita falsi positivi evidenti.
- Aggiungi un hint pratico di remediation per ciascuna regola.

Log di input (<logs>):
<logs>
{{LOG_GREZZI}}
</logs>
```

**Perché Funziona**
- **Schema operativo:** pronto per pipeline di alerting.
- **Soglie e finestre:** supportano tuning e SLO.
- **Remediation hints:** utili per tempo di risoluzione.

**Fonti**
- xAI/Grok — linee guida generali di prompting e best practice operative.
- Pattern SRE per definizione di alert da log.
